# The Trio — Landing Page UI/UX & Storytelling Specification
### React Frontend · Complete Implementation Guide

---

## 0. Document Purpose

This spec is written for an AI agent building a React frontend.
It covers: narrative arc, section-by-section copy + UX intent,
interaction design, visual design system, component behavior,
and motion rules. Read it top to bottom before writing a single line.

---

## 1. Design Philosophy & Aesthetic Direction

### 1.1 The Tone: Refined Confidence

The Trio is not a startup. It is not a big agency. It is three people
who are better than most teams of twenty, and they know it — without
performing it. The aesthetic must feel like that.

Reference points: Pentagram's website. Linear's product site.
A good hospital's admission form. A good law firm's letterhead.
**Nothing flashy. Everything precise.**

The one thing a visitor must remember: *these people think before they build.*

---

### 1.2 Aesthetic Decision

**Direction: Editorial Minimalism with Kinetic Typography**

- Dark background. Off-white text. One accent color.
- Oversized, deliberate typographic moments at section entrances.
- Every animation is purposeful — it reveals something, it doesn't decorate.
- Whitespace is used with intent, not default. Sections breathe.
- No stock imagery. No hero illustration. No gradients as decoration.
  Geometry only. Type only. Motion only.

**The metaphor that drives every design decision:**
A surgeon's table. Nothing unnecessary. Everything at hand. Total control.

---

### 1.3 Color System

```css
:root {
  /* Backgrounds */
  --bg-primary:      #0A0A0A;   /* Near-black — primary canvas */
  --bg-surface:      #111111;   /* Slightly lifted surface */
  --bg-card:         #161616;   /* Card/module backgrounds */
  --bg-border:       #222222;   /* Borders and dividers */

  /* Typography */
  --text-primary:    #F0EDE6;   /* Warm off-white — main text */
  --text-secondary:  #888880;   /* Muted — supporting text */
  --text-ghost:      #333330;   /* Very muted — decorative/numbers */

  /* Accent — used sparingly, maximum impact */
  --accent:          #C8A96E;   /* Warm gold — the one color that matters */
  --accent-dim:      #7A6340;   /* Dimmed gold — hover states */

  /* Functional */
  --success:         #4A7C59;
  --error:           #8B3A3A;
}
```

**Rules:**
- The gold accent appears on: active states, key headings that need
  weight, the single CTA button, hover underlines.
- Never use it decoratively. Use it to mark importance.
- No gradients except a single subtle radial on the hero section.

---

### 1.4 Typography System

```
Display font:  "DM Serif Display" (Google Fonts)
               — Used for section titles, pull quotes, hero heading
               — Gives intellectual weight, editorial authority

Body font:     "DM Mono" (Google Fonts)
               — Used for body text, labels, metadata
               — Signals precision, technical credibility
               — Unexpected choice for body — creates character

Accent font:   "DM Serif Display" italic
               — For pull quotes and the narrative callouts
```

**Why DM Mono for body:** Most consultancy sites use a humanist sans
(Inter, Neue Haas, etc). DM Mono reads as a team that thinks in code —
it implies specificity, rigor, and technical honesty. The serif display
for headings balances it with intellectual weight.

```css
/* Scale — use only these sizes */
--text-hero:   clamp(56px, 8vw, 96px);
--text-title:  clamp(32px, 4vw, 52px);
--text-section:clamp(22px, 2.5vw, 32px);
--text-body:   16px;
--text-small:  13px;
--text-label:  11px; /* letter-spacing: 0.12em; text-transform: uppercase; */

/* Line heights */
--lh-display:  1.05;
--lh-title:    1.15;
--lh-body:     1.75;
```

### 1.5 Motion System

```
Philosophy: Reveal, don't decorate.
Every animation answers: "what does the user now understand
that they didn't before this motion?"

Durations:
  --dur-instant:   80ms   (feedback: button clicks)
  --dur-fast:      200ms  (hover transitions)
  --dur-standard:  400ms  (section reveals, fades)
  --dur-slow:      700ms  (hero entrance, large type reveals)
  --dur-cinematic: 1200ms (staggered sequences only)

Easing:
  --ease-out:  cubic-bezier(0.16, 1, 0.3, 1)   (entries — fast start, soft settle)
  --ease-in:   cubic-bezier(0.4, 0, 1, 1)       (exits)
  --ease-snap: cubic-bezier(0.34, 1.56, 0.64, 1) (interactive elements, slight overshoot)

Rules:
  - All scroll-triggered animations use IntersectionObserver.
  - Use Framer Motion for React. Do not use CSS @keyframes for complex
    sequences — use Framer's variants system for choreography.
  - Respect prefers-reduced-motion: wrap all motion in a useReducedMotion check.
  - No looping animations on the visible page. Motion should stop when complete.
    (Exception: the hero section cursor element — one subtle pulse, then stop.)
```

## 2. Page Architecture — The Narrative Arc

The page is a story. The visitor is the protagonist.
They arrive with a problem. They leave with a solution.

```
ACT 1 — THE HOOK
  Section 1: Nav (persistent)
  Section 2: Hero — "The Statement"

ACT 2 — THE PROBLEM (mirror the client's reality back to them)
  Section 3: "Most Teams Build the Wrong Thing"

ACT 3 — THE SYSTEM (what the Trio actually does)
  Section 4: "How It Works" — the 5 stages
  Section 5: "The Intelligence" — the AI system

ACT 4 — THE PEOPLE (who they actually are)
  Section 6: "The Trio" — the three profiles

ACT 5 — THE PROOF (grounding)
  Section 7: "Grounded In" — the framework wall

ACT 6 — THE INVITATION
  Section 8: "Book a Session" — the single CTA
  Section 9: Footer
```

**Scroll behavior:** Single-page. All sections stacked vertically.
Standard scroll — no scroll-jacking. Let the browser scroll naturally.
Use scroll-triggered reveals for content. Do NOT use forced full-screen
snapping — it fights the user.

## 3. Section-by-Section Specification

---

### SECTION 1: Navigation

**Type:** Persistent sticky top bar

**Behavior:**
- Transparent at page load
- On scroll past 80px: background becomes `--bg-surface` with
  `backdrop-filter: blur(12px)` and a 1px bottom border (`--bg-border`)
- Smooth transition: 300ms ease

**Layout:**
```
[  TRIO  ]                    [  Book a Session →  ]
  wordmark                         CTA button
```

**Wordmark:** "TRIO" in DM Serif Display, 18px, `--text-primary`.
No tagline in nav. Just the name.

**CTA Button:**
- Border: 1px solid `--accent`
- Background: transparent
- Text: "Book a Session →" in DM Mono, 12px, `--accent`
- On hover: background fills with `--accent`, text becomes `--bg-primary`
- Transition: `--dur-fast`

**No other nav links.** The page is a single narrative — no menu
breaks the story. The only decision the visitor makes is: scroll or book.

---

### SECTION 2: Hero — "The Statement"

**Storytelling intent:**
State the position before explaining it. The client should feel
slightly challenged — "wait, say that again" — and keep scrolling.

**Layout:** Full viewport height. Vertically centered text. Left-aligned.

**Content:**

```
[label — small caps, --text-secondary]
Product Intelligence & Execution

[hero heading — DM Serif Display, --text-hero]
You bring the insight.
We deliver the
right product.

[body — DM Mono, 16px, --text-secondary, max-width: 480px]
A few structured conversations with the Trio and the system
does the rest — research, validation, prototype, and a product
direction you can trust.

[CTA]
[ Book a Session → ]    [ See How It Works ↓ ]
  (primary button)          (ghost link)
```

**Visual treatment:**
- Subtle radial gradient centered on heading area:
  `radial-gradient(ellipse 60% 50% at 30% 50%, rgba(200,169,110,0.05) 0%, transparent 70%)`
- No image. No illustration. Type is the hero.
- A single thin horizontal rule below the body text, 80px wide, `--accent`.
  Animates in after the text: draws from 0px to 80px width in 600ms.

**Entrance animation:**
```
1. Label fades up (0 → 1 opacity, 0 → -12px translateY) — 0ms delay
2. Line 1 of heading fades up — 100ms delay
3. Line 2 of heading fades up — 200ms delay
4. Line 3 of heading fades up — 300ms delay
5. Body paragraph — 450ms delay
6. Buttons — 600ms delay
7. Gold rule draws in — 700ms delay
Total entrance: ~900ms, feels deliberate not rushed
```

**Scroll indicator:** At bottom of hero, a small downward chevron
in `--text-ghost`. Disappears after scrolling 20% of viewport height.

## 4. Responsive Design Rules

**Breakpoints:**
```
mobile:  < 640px
tablet:  640px – 1024px
desktop: > 1024px
```

**Section-specific responsive rules:**

| Section | Desktop | Tablet | Mobile |
|---|---|---|---|
| Hero | Left-aligned, 50% width heading | Left-aligned, 70% width | Full width, centered |
| Problem | 2-column | 2-column | Stacked |
| Stages | 5 cards horizontal | 3+2 grid | Stacked vertical list |
| System | 2-column | 2-column | Stacked, schema hidden |
| Trio | 3-column | 3-column | Stacked |
| Frameworks | 3×2 grid | 2×3 grid | 1×6 stacked |
| CTA | Centered 60% width | Centered 80% | Full width |

**Typography scaling:** Use `clamp()` for all display and title sizes.
Body text stays at 16px on all breakpoints — do not reduce it on mobile.

## 5. Component Specifications

### 5.1 ScrollReveal Component

Wrap all scroll-triggered content in this behavior:

```jsx
// Behavior spec — implement with Framer Motion
// Initial state: opacity: 0, y: 24
// Animate to: opacity: 1, y: 0
// Trigger: when element is 15% inside the viewport
// Duration: 0.5s ease-out
// Once: true (does not re-trigger on scroll up)
```

### 5.2 Stage Card (Process section)

```jsx
// Hover: border-color → --accent, translateY(-4px)
// Transition: 200ms ease-out
// Gold connector dot between cards — 6px circle, --accent fill
```

### 5.3 Nav Component

```jsx
// useScrollPosition hook: triggers at 80px scroll
// Transition: background opacity 0→1, backdrop-filter 0→blur(12px)
// Duration: 300ms
```

### 5.4 Booking Modal

```jsx
// Trigger: CTA button click
// Open animation: scale(0.96)→scale(1), opacity 0→1, 200ms ease-out
// Backdrop: rgba(0,0,0,0.7), blur(4px)
// Close: Escape key, backdrop click, close button (top-right ×)
// Focus trap inside modal when open (accessibility)
// Form validation: inline, on submit. No per-field validation while typing.
```

### 5.5 Tag Pills (Trio section)

/* Tag pill */

## 6. Accessibility Requirements

- All interactive elements: visible focus ring (2px `--accent` outline, 2px offset)
- Color contrast: `--text-primary` on `--bg-primary` → WCAG AA minimum
- Booking modal: focus trap, escape to close, aria-modal="true"
- `prefers-reduced-motion`: all animations disabled / reduced to simple opacity fade
- All section headings: proper `h1/h2/h3` hierarchy (h1 = hero only)
- Images: none used. No alt text needed.
- Booking form: all inputs labeled, errors announced via aria-live

## 9. The One Test

When the implementation is complete, apply this test:

Open the page. Scroll from top to bottom in 90 seconds.
Ask: *do I understand what these people do, why they are different,
and why I would trust them — without having read a single word carefully?*

If yes: the design is working.
If no: the visual hierarchy is broken — fix it before the copy.

The page should be comprehensible at the level of layout alone.
Copy makes it credible. Design makes it undeniable.

On the aesthetic direction — editorial minimalism with kinetic typography
In 2025, minimalist landing pages embraced softer, subdued color schemes — monochromatic tones that reduce visual noise and enhance focus on essential elements like CTAs and headlines. Fabiofumarola But the research also shows the competitive signal: the brands that win in 2025 are not the ones with the fanciest graphics — they are the ones who understand their audience on a deeper level: how they think, how they scroll, and what makes them pause. @hgraca The Trio's audience is technical and skeptical. They do not pause for decoration. They pause for precision. Hence: dark background, DM Mono body font, gold accent used once per section.
On the narrative architecture — scroll as chapter structure
Scrollytelling achieves up to 400% higher engagement than static content by using scroll position to control story progression, trigger animations, and reveal content progressively. Harvard Business Review The spec structures the page as six acts precisely because scrollytelling allows you to control the narrative — unveiling information at just the right moments to build curiosity and maintain engagement. Fullstory The visitor's job is to keep scrolling because each section resolves the tension of the one before it.
On the copy's challenge-first approach
Effective software development landing pages frame benefit around growth rather than merely staffing — and every element should be a functional demonstration of attention to detail and UX philosophy. UXPin The "Most Teams Build the Wrong Thing" section does this: it mirrors the client's current pain before offering the alternative, which is the structural pattern that converts readers into believers.
On the booking form's radical simplicity
Great landing pages combine clarity, purpose, and performance — guiding the visitor through a journey and removing friction so people feel confident taking the next step. GitHub A single-page modal with four fields and no multi-step flow is the execution of that principle. The Trio does not need a 12-question onboarding form. They need context + intent. Four fields covers it.
The three decisions that give the spec its character: DM Mono as body font (signals technical rigor, breaks convention), the ASCII pipeline schema in the system section (shows the process without an animation library), and the footer line "Built with the same system we sell" (closes the loop on the whole philosophy without saying it directly).
